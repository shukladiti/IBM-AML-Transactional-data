================================================================================
  WHY ACCURACY IS NOT REPORTED
================================================================================

  In AML fraud detection the label distribution is severely imbalanced —
  roughly 1 fraud per 100 transactions. A model that labels every single
  transaction as "not fraud" achieves 99% accuracy while catching zero frauds.
  Accuracy rewards the majority class and is meaningless here.

  Metrics used instead:

  AUC-PR   Area under the Precision-Recall curve. The primary metric.
           Measures how well the model ranks fraud above legitimate
           transactions across all thresholds. Robust to class imbalance
           because it focuses entirely on the positive (fraud) class.

  ROC AUC  Overall discrimination between classes. Less sensitive to
           imbalance than accuracy but can be overly optimistic when
           negatives vastly outnumber positives.

  KS       Maximum separation between cumulative fraud and legitimate
           score distributions. KS = 0.89 means very clean separation.

  Precision / Recall at operational thresholds — the business-facing
           metrics. Precision = fraction of flagged transactions that are
           actually fraud. Recall = fraction of all fraud that gets flagged.

================================================================================
  TEST SET DESIGN
================================================================================

  The test set uses a production-like fraud rate of 1%, enforced by
  downsampling the majority class. Evaluating on a 50/50 stratified split
  would inflate precision metrics that would never hold in production.

  The test window covers the most recent 120 days, time-ordered and fully
  held out from training. The train set uses stratified oversampling (~2%
  fraud rate) so the model sees enough positive examples during learning.

================================================================================
  KNOWN LIMITATIONS
================================================================================

  Early stopping leakage
    n_estimators is selected by evaluating against the holdout set, which
    is a mild form of data leakage. Fix: carve a validation split from the
    training period for early stopping only. Metrics may be slightly
    inflated as a result.

  Counterparty feature fragility
    Described above. Production monitoring should include PSI tracking on
    REPEAT_COUNTERPARTY_COUNT. PSI above 0.5 should trigger review and
    potential emergency retraining.

  Sampling non-determinism
    Snowpark sample() has no random seed. Each run trains on a slightly
    different subset, so reported metrics include sampling variance in
    addition to model variance. Observed AUC-PR variance across valid
    runs: 0.672 - 0.685.
